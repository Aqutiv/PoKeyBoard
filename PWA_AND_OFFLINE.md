# PWA and offline

## Manifest

Generated by vite-plugin-pwa from `vite.config.ts`: name/short_name **PoKeyBoard**, `display: standalone`, stable `id`, theme `#1d1a17` on background `#171412`, standard + maskable icons (from `assets/branding/pokeyboard-icon-master.png` via `scripts/build-icons.mjs`) and an Apple touch icon linked in `index.html`. `start_url`, `scope`, and `id` all derive from the single configurable base path (`POKEYBOARD_BASE`).

## Service worker (src/pwa/service-worker.ts, Workbox injectManifest)

- **Precache:** the app shell only â€” JS/CSS/HTML/icons, ~1.2 MB. Piano samples are explicitly excluded from precache.
- **Navigation fallback:** every navigation serves the precached shell, so the app starts offline. Hash routing keeps this to a single URL.
- **Runtime caching:** `**/piano/**` â†’ Cache First into `pokeyboard-piano-samples-v2` (immutable versioned filenames; superseded generations listed in `STALE_PIANO_SAMPLE_CACHES` are deleted on service worker activation). The same named cache backs Settings â†’ **Download piano for offline use** (progress by manifest bytes) and **Delete downloaded samples** â€” deleting samples never touches takes, and the cache purges itself first under quota pressure so IndexedDB takes survive.

## Updates

A new deploy makes the browser install a **waiting** worker; `updateManager` (workbox-window) surfaces "update available", and activation happens only when the user applies it in Settings â€” the button is disabled while recording/playing/exporting, and nothing ever auto-reloads mid-session. Applying sends `SKIP_WAITING` and reloads on `controlling`.

## Installation

`beforeinstallprompt` is captured where browsers fire it (Settings shows an Install button; a dismissal starts a 14-day quiet period). iOS has no programmatic prompt: Settings shows Share â†’ **Add to Home Screen** instructions and warns that the installed app may use a **separate storage context** â€” export a JSON backup before switching, and prefer the installed icon for important takes.

## Caching headers (host configuration)

| Path                        | Header                                               |
| --------------------------- | ---------------------------------------------------- |
| `/index.html`               | `Cache-Control: no-cache`                            |
| `/service-worker.js`        | `Cache-Control: no-cache`                            |
| `/manifest.webmanifest`     | `Cache-Control: no-cache` (or short max-age)         |
| `/assets/*` (hashed js/css) | `Cache-Control: public, max-age=31536000, immutable` |
| `/piano/*` (versioned pack) | `Cache-Control: public, max-age=31536000, immutable` |
| `/icons/*`                  | `public, max-age=86400`                              |
